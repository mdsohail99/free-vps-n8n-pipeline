# 📖 Step-by-Step Deployment & Configuration Guide

This document provides a comprehensive, beginner-friendly walkthrough to set up your Always-Free AI Automation Server from scratch.

---

## 📋 Phase 1: Account Setup & Prerequisites

Before executing any commands on your server, make sure you have created free accounts and collected your API tokens:

### 1. Oracle Cloud Infrastructure (OCI)
* **Goal**: Get a free cloud VPS.
* **Action**: Sign up at [Oracle Cloud Free Tier](https://cloud.oracle.com/). You will need a credit/debit card for verification. Select a home region nearest to your audience.

### 2. Notion Developer Account
* **Goal**: Get access to Notion's database API.
* **Action**:
  1. Register or log in at [Notion my-integrations](https://www.notion.so/my-integrations).
  2. Click **+ New integration**. Select the workspace where your database will reside.
  3. Copy your **Internal Integration Token** (e.g. `ntn_...`).
  4. Create or choose a parent page in Notion where you want the database created.
  5. Click `...` at the top right of the parent page → **Connections** → add your new Integration connection.

### 3. OpenRouter API Key
* **Goal**: Get free LLM endpoints (like the NVIDIA Nemotron Series).
* **Action**: Sign up at [OpenRouter](https://openrouter.ai/), go to settings, and generate a new API key (`sk-or-v1-...`).

### 4. Slack Webhook (Optional)
* **Goal**: Get real-time morning briefings on Slack.
* **Action**: Create an app in your Slack workspace and enable **Incoming Webhooks**. Copy the webhook URL.

## 🖥️ Phase 2: Provisioning the VPS Host

1. Go to your **Oracle Cloud Console**.
2. Navigate to **Compute → Instances → Create Instance**.
3. Configure the following parameters:
   * **Image**: `Ubuntu 24.04` (Canonical).
   * **Shape**: `VM.Standard.A1.Flex` (Ampere ARM). Allocate `4 OCPUs` and `24 GB of RAM` (Always Free eligible).
   * **Networking**: Create a new Virtual Cloud Network (VCN) and assign a public IP.
   * **SSH Keys**: Download/Save the private key (`.key` or `.pem`) generated by Oracle to your local computer.
4. **Configure Security Lists (Ports 80 & 443 Ingress)**:
   * Go to **Virtual Cloud Networks → [Your VCN] → Security Lists → Default Security List**.
   * Click **Add Ingress Rules**.
   * Add rules allowing TCP traffic on ports **`80`** and **`443`** from source `0.0.0.0/0`.

## 🔑 Phase 3: First SSH Connection & Deployment

1. Open your local terminal (Terminal on macOS/Linux, or PowerShell on Windows).
2. SSH into your VPS:
   ```bash
   ssh -i /path/to/your-key.key ubuntu@<YOUR_VPS_PUBLIC_IP>
   ```
3. Update system packages, clone the repository, and bootstrap the server:
   ```bash
   git clone https://github.com/mdsohail99/free-vps-n8n-pipeline.git
   cd free-vps-n8n-pipeline
   bash vps_setup.sh
   ```
   *This script installs Docker, tunes host firewall parameters, and configures swap space.*
4. Exit the SSH session and connect again so Docker group permissions apply:
   ```bash
   exit
   ssh -i /path/to/your-key.key ubuntu@<YOUR_VPS_PUBLIC_IP>
   ```

## 🗄️ Phase 4: Configure Credentials & Notion Database

1. Navigate to the pipeline directory:
   ```bash
   cd ~/free-vps-n8n-pipeline
   ```
2. Copy the example config and edit:
   ```bash
   cp .env.example .env
   nano .env
   ```
   *Populate `NOTION_INTEGRATION_TOKEN`, `NOTION_PARENT_PAGE_ID`, `SLACK_WEBHOOK_URL`, `N8N_HOST` (e.g. `n8n.yourdomain.com`), and `WEBHOOK_URL` (e.g. `https://n8n.yourdomain.com/`).*
3. Execute the Notion schema builder to auto-create your database page:
   ```bash
   python3 setup_notion_db.py
   ```
   *This script outputs your new `NOTION_DATABASE_ID`. Add this ID to your `.env` file.*
4. Launch the Docker Compose stack:
   ```bash
   docker compose up -d
   ```

## 📥 Phase 5: Importing & Configuring n8n Workflows

1. Open your browser and navigate to your domain (e.g. `https://n8n.yourdomain.com`). Set up your admin account.
2. In n8n, click **Workflows → Import from File** and select `ai_daily_briefing_workflow.json` from your local machine.
3. **Configure Node Credentials** inside n8n:
   * **Notion API**: Create a `Header Auth` credential named `Notion API`. Set name to `Authorization` and value to `Bearer <your Notion token>`.
   * **OpenRouter API**: Create a `Header Auth` credential named `OpenRouter API`. Set name to `Authorization` and value to `Bearer <your OpenRouter API key>`.
   * **VPS SSH**: Create an `SSH Private Key` credential named `VPS SSH`. Enter `ubuntu` as user, your public IP as host, and paste the contents of your VPS private SSH key.
4. Open nodes `Node 6`, `Node 8`, `Node 14`, and `Node 16` and ensure they are connected to the correct credentials in the n8n UI.
5. Click **Save** and toggle the workflow to **Active**!
